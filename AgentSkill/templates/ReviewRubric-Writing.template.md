# Review Rubric（Writing）— 评分与整改模板

> 注意：本模板仅提供参考结构，不是执行规范；实际输出必须遵循本 skill 的 `SKILL.md` 与对应阶段的 `stages/*/SKILL.md`。模板与规则冲突时，以规则为准。

标度：0–10 分（允许小数）

锚点解释：
- 9–10：优秀；结构清晰、覆盖充分、风格贴合、图示有效、引用可靠
- 7–8：良好；可发布，少量润色即可
- 5–6：可读但薄弱；结构/覆盖/例子/图示存在明显缺口
- 3–4：问题较大；读者难以按文档完成目标
- 0–2：不可用；事实错误或来源不清

---

## 1. 硬门禁（先过门禁再谈分数）

- **过程合规违规（Stop-the-line）**
  - 场景：未授权就写作/修改交付物（Execution Authorization!=received 但发生写作产出）、Plan/Task 契约缺失或残留大量占位符、关键改动无法映射到 Task 条目、或“可运行示例/来源”门禁被跳过
  - 结论必须为：`BLOCKED`
  - 动作：回到 Plan 补齐 Plan/Task/State 与证据门禁，再重新走授权门禁

- **验证失败（failed）**（例如：关键事实无来源、术语前后矛盾、承诺可运行但示例不可运行）
  - 结论必须为：`BLOCKED`
  - 动作：必须返工补齐证据/修复 → 再复评

- **验证缺失（missing）**（例如：示例不可在当前环境运行、部分来源暂不可获得）
  - 结论默认应为：`NEEDS_USER_DECISION`
  - 动作：必须提醒风险；是否返工补齐由用户决定（默认建议补齐最小可验证证据）

## 3. 总分与结论

- 总分（建议取均值或加权均值）：

结论枚举：
- `PASS`：验证通过且总分 ≥ 7.0/10
- `NEEDS_USER_DECISION`：
  - 验证通过但总分 < 7.0/10（需提醒风险；是否返工由用户决定）
  - 或验证缺失（missing）：必须提醒风险；是否返工由用户决定
- `BLOCKED`：验证失败（failed）（必须返工并复评）

> 用户偏好规则（要执行）：
> - 验证不通过 → 必须复评
> - 总分 < 7.0/10 且验证不通过 → 必须返工并复评
> - 总分 < 7.0/10 但验证通过/或无验证可做 → 必须提醒用户；是否返工复评取决于用户反馈
